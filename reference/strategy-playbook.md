# 投资策略总纲（Strategy Playbook）

激进型场外基金投资的完整策略梳理。**目标：规则内收益最大化**。决策权威只有 `decide.py` 引擎——本文是给你（顾问）理解策略全貌、把引擎输出讲清楚的总纲，不是另一套可绕过引擎的规则。改规则必须先过梦境实验室（见 `analysis-and-backtest.md`）。

## 一、投资哲学与目标

- **收益最大化优先**：激进策略，偏好股票型/指数型/行业主题基金。风控闸门只为保住本金、服务长期复利，不是束缚。
- **让利润奔跑**：盈利标的（尤其趋势 QDII 指数如 006479）不机械止盈，跌破止损再走（`take_profit_policy.mode=off`，回测证据：牛市 +73.98% vs 基线 +47.27%）。
- **不追一日脉冲**：任何买入都要看 7日/30日/6月的真实趋势 + 典型新闻，区分「真强势」和「超跌反弹（下跌中继）」。
- **不固守一篮子**：跳出固定持仓，先扫板块再下钻选基，跨赛道分散。

## 二、决策优先级（引擎 `_evaluate_rules` 执行顺序）

1. **账户级回撤保护** — 组合从峰值回撤 ≥10% → 所有买入降级为观察。
2. **卖出（对每只持仓，优先级从高到低）**：紧急止损 → 趋势破位退出（200日线）→ RSI 超买减仓 → 分层止盈（当前关闭）。
3. **总仓位超上限** → 回撤最大持仓（`position_cap_trim`）。
4. **买入（对每只候选）**：低吸（当日跌 >3%）→ 信号买入（RSI/突破，当前禁用）。
5. **分批建仓** — 总仓位低于 regime 目标下限且 HS300 在 200 日线上方 → 按**多窗口动量**挑候选建仓。
6. **限购裁剪** — 对所有买入按 `fund_constraints` 裁金额（如 006479 ¥10/天）。
7. **份额标注** — 给每条买入加 `horizon` + `share_class`（短 C 长 A）。

规则 ID 速查见 `rule-and-fund-reference.md`。

## 三、市场环境与仓位框架

引擎用 HS300 近 20 日涨跌分三档（`market_regime.label`）：

| 环境 | 判定 | 总仓位上限 | 目标下限 | 单只上限 |
|------|------|-----------|---------|---------|
| 牛市 | HS300 20d > +5% | 95% | 70% | 30% |
| 震荡市 | -5% ~ +5% | 85% | 50% | 25% |
| 熊市 | < -10% | 60% | 30%（禁新建仓） | 15% |

`portfolio_advice` 每次报告必展示：当前仓位 vs 目标区间 + 可部署现金。仓位低于下限 → 引擎自动分批建仓（每批最多 2 只、留 10% 现金线）。

## 四、选基纪律：板块下钻 + 多窗口

**别只盯固定那几只。** 标准下钻流程：

1. `fetch_fund.py sector-scan --top 8` — 看哪些板块是**多窗口真强势**（今日/7日/30日/6月同向上行），剔除「超跌反弹·谨慎」「弱势下行」。
2. `fetch_fund.py discover --sector <强势方向>` — 从该方向挑场外基金，按多窗口一致性评分（近6月×0.5 + 近3月×0.3 + 近1月×0.2，一周飙太多扣分防追高），已排除持仓。
3. `decide.py run --discover N` — 把发现的新候选并入引擎候选池，引擎按多窗口动量（`_candidate_momentum`：20/60/120 日加权 + 一致性加成）择优建仓。
4. WebSearch 该方向政策/产业新闻佐证，再定仓位。

引擎候选池 = 持仓 + 观察池（`simulate.DEFAULT_FUNDS`）+ `--discover` 注入的新发现。**自动 cron 路（daily_report）只展示发现的新方向、不自动买**（避免每日轮换追新），新方向的买入交人工结合短C长A判断。

## 五、份额类别：短期买 C、长期买 A

引擎给每条买入标 `share_class.preferred`：

- **短线**（低吸 `low_buy`、信号买 `rsi_oversold_buy`/`momentum_breakout`）→ **C 类**：无申购费、按日计销售服务费，持有约 <1~2 年总费更低，适合波段。
- **长线**（核心建仓 `position_build`）→ **A 类**：有申购费但无销售服务费，长期持有总成本更低。

若持仓/候选当前份额 ≠ 推荐类别，`fetch_fund.py share-class <code> --prefer C|A` 查对应兄弟份额代码，买那个份额、记那个代码。（如 006479 是纳指 C 类，若要长期核心持有应改买 A 类 270042。）

## 六、风控闸门（拦截在 `blocked_actions[]`）

现金储备 ≥10% · 单只 ≤25%（牛市 30%）· 单赛道上限（科技 50%/消费 30%/…）· 禁追涨（5日 >10%）· 熊市禁新建仓 · 大盘数据缺失暂停建仓。止损：单日 >7% 或 3日 >10% 减半、亏损 >20% 清仓、按持有期分档止损、200日线破位减半。**自主 ≠ 失控**——这些闸门照常生效。

## 七、复盘闭环：每次操作前回溯、操作后记忆

- **操作前回溯**：`decide.py review --summary`（历史择时踩中/套牢记忆）+ `decide.py stats`（各规则胜率/期望）+ `db.py trades`。哪些规则常踩中就更信，常套牢就更谨慎。
- **操作后记忆**：`decide.py review --save` 在交易满 horizon 天后用事后净值判定择时（踩中/追高套牢/规避下跌/卖飞），写入 `trade_reviews`，沉淀为经验。盘尾日报的操作时间线自动带复盘点评。
- **策略进化**：`decide.py stats` 找出期望为负的规则 → 梦境实验室在 ≥2 个不同市况窗口验证 → `strategy_lab.py --promote` 晋升新决策树版本。
